Physical Requirements

The physical structure of the system is described below:

  1. Propose Solutions → CONOPs
    • MOEs + KPPs
    • Risks
    • TRLs + IRLs
    • Cost Estimate
    • Requirements + Verification Plans
  2. Select a Solution Based on System-Level MOEs
  3. Develop Component-Level Performance Requirements
  4. Select Components Using Trade Studies
    • Micros/processors
  5. Simulate
    • Select Outputs Based on MOEs
    • Select Relevant Inputs
  6. Select Tech Stack Based on Micros + Performance Requirements
  7. Generate State Machine
  8. Generate Algorithms from Necessary UC Narratives
  9. Generate Unit Tests for Embedded Code
  10. CAD
  11. Schematics
  12. PCBs
