Non-Functional Requirements

Non-functional requirements specify criteria that judge the operation of a system, rather than specific behaviors. These include:

  • Usability: The system should be intuitive and easy to use for all user roles.
  • Security: All data must be encrypted in transit and at rest.
  • Maintainability: The system should allow updates and patches with minimal downtime.
  • Portability: The system must run on Windows, macOS, and Linux platforms.
  • Compliance: The system must adhere to GDPR and ISO 27001 standards.
